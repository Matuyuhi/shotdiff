## 2023-10-27 - Rust Iterator and Math Optimizations in Pixel Processing
**Learning:** In Rust image processing, iterating over `.pixels()` creates wrapper structs for every pixel, which creates massive overhead when compared to chunking raw slices `as_flat_samples().as_slice().chunks_exact(4)`. Additionally, integer arithmetic `(19595 * r + 38470 * g + 7471 * b) >> 16` instead of floating point math for calculating luma provides a 2x-3x performance speedup. Lastly, unrolling small loop constructs manually like `a[0].abs_diff(b[0]).max(a[1]...` helps LLVM auto-vectorize and avoids loop index branching.
**Action:** When working on tight loops involving image processing in Rust: Use raw slice iteration instead of struct abstraction iterators, prefer bitwise integer math over floating points for approximations (like RGB to luma), and unroll tight per-pixel loops manually.
